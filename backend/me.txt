Delivery philosophy
Athar builds by writing the system down before writing code. Every serious product starts with product and technical documents so scope, architecture, data models, integrations, acceptance criteria, and safety gates are explicit.

His standard delivery stack includes:
- Problem definition document.
- Product Requirements Document (PRD).
- Technical Requirements Document (TRD).
- Architecture document.
- Data schema.
- Phase plan with acceptance criteria.
- Agent governance rules such as CLAUDE.md / AGENTS.md.
- COMMITS.md audit trail.
- Pre-deployment safety checklist.
- Handoff documentation and next-phase roadmap.

How Athar uses AI
Athar uses AI coding agents as scoped implementation partners, not unsupervised product owners. He remains the architect, reviewer, and quality gate.

He defines the system design, scope, data model, and task files first. Agents such as Claude Code, Codex, Gemini CLI, and Cursor are used to execute tightly scoped implementation work. Agent tasks are expected to stay within a written spec. They should not alter architecture, authentication, payments, production configuration, security rules, or deployment behavior without explicit instruction.

Every meaningful change should be reviewed against the spec, tested, and recorded. Typecheck, build, and relevant tests should pass before a change is treated as shippable.

Pre-deployment safety principles
Athar does not treat launch as just a deploy button. Before production release, products should be checked for:
- Authorization and ownership checks, especially IDOR/BOLA risk.
- Short-lived, single-use password reset links.
- Input validation against SQL injection, NoSQL injection, XSS, SSTI, unsafe eval, unsafe rendering, and unsafe shell execution.
- Production CORS locked to approved domains.
- Rate limiting on login, signup, reset, upload, search, AI, payment, and expensive endpoints.
- Safe error handling with no stack traces, internals, file paths, secrets, or debug pages exposed to users.
- Database indexes for hot queries.
- Structured logging, monitoring, health checks, and alerting.
- Rollback strategy before deployment.
- App Store privacy, metadata, permission strings, subscription/IAP flow, and data deletion checks for mobile products.
